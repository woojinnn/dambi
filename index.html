Dambi Privacy Policy

Dambi 개인정보처리방침 (Privacy Policy)

시행일: 2026년 6월 22일

Dambi(“본 확장 프로그램”)는 Web3 지갑 사용자를 위한 서명 전 보안 분석 브라우저 확장 프로그램입니다. 트랜잭션·서명·거래소 주문이 서명되기 직전에 그 내용을 정적으로 디코드(트랜잭션 시뮬레이션 없음)하고, 사용자가 설정한 정책으로 평가해 Pass / Warn / Fail 판정을 제공합니다. 판정은 사용자의 브라우저 안(번들된 정책 엔진)에서 내려지며, 외부 서버가 판정을 대신 결정하지 않습니다. 로그인한 경우에는 제재·시세·평판 같은 실시간 위험 정보를 선택적으로 함께 활용합니다. 본 방침은 Dambi가 어떤 데이터를 다루고, 무엇을 다루지 않는지 설명합니다.

1. Dambi가 접근하지 않는 정보

  • 개인키·시드 문구에 접근하지 않습니다. Dambi는 사용자를 대신해 서명·전송·체결하지 않으며, 자금을 옮기지 않습니다. 판정만 제공하고, 최종 결정은 사용자와 지갑이 내립니다.
  • 페이지 콘텐츠를 수집하지 않습니다. Dambi는 페이지의 DOM(본문·제목), 쿠키, 로컬·세션 스토리지, 입력 필드(비밀번호 등)를 읽지 않습니다. 평가를 위해 다루는 것은 아래 §2의 지갑 요청과 사이트 식별 정보뿐이며, 그 구체적 범위와 처리 방식은 §2에 설명합니다.

2. Dambi가 페이지에서 가로채는 것 (분석 대상)

Dambi는 서명·전송 직전에 평가하기 위해, 페이지가 보내는 지갑 요청을 가로챕니다(아웃바운드 요청 모니터링):

  • 지갑 RPC 요청 — eth_sendTransaction(트랜잭션 요청 객체), eth_signTypedData*(EIP-712 서명 데이터)
  • HyperLiquid 거래소 주문 — 사이트가 보내는 /exchange 요청(fetch/XHR 인터셉트)

가로챈 트랜잭션 요청 객체에는 보내는 주소·받는 주소·calldata·전송액과, 사이트가 함께 넣은 표준 트랜잭션 파라미터(gas, nonce 등)가 그대로 포함됩니다. 이는 정확한 분석에 필요한 범위이며, 페이지 본문이나 §1의 정보는 포함하지 않습니다. Dambi가 사이트 식별을 위해 외부로 전송하는 것은 호스트명(예: app.uniswap.org)뿐입니다. 페이지의 전체 URL(경로·쿼리 포함)은 어느 거래소(venue) 페이지인지 로컬 판별에만 잠시 사용되고 수집·전송되지 않으며, 페이지 제목은 읽지 않습니다. (HyperLiquid 주문 평가 시 호출되는 거래소 공개 API 엔드포인트 주소는 §4의 위험 조회에 함께 쓰이나, 이는 사용자가 보고 있는 페이지 URL이 아니라 거래소의 공개 API 주소입니다.)

3. 기기 안에서 처리되는 정보

정책 평가(Cedar 정책 엔진)는 사용자의 브라우저 안에서 로컬로 실행됩니다. 판정 자체를 외부 서버가 대신 내리지 않으며, 정책 정의와 판정 결정은 기기를 벗어나지 않습니다. 판정 기록·실행 기록·진단 로그는 사용자 기기의 chrome.storage.local(확장 전용 저장소, 웹페이지 접근 불가)에만 저장되며, 본 확장 프로그램이 이를 외부로 전송하지 않습니다. 제3자 애널리틱스나 행동 추적(analytics/telemetry)을 사용하지 않습니다.

4. 외부로 전송되는 정보

거래 의도를 해석하고 위험을 평가하기 위해 다음 데이터가 외부로 조회됩니다.

  • 레지스트리·토큰 메타데이터 조회 (로그인 무관, 항상): 거래 대상 컨트랙트 주소, 함수 선택자(selector), 토큰 컨트랙트 주소를 Dambi 레지스트리 서버에서 조회합니다. 로그인 여부와 관계없이 평가 중 발생하며, 이 조회에는 사용자의 지갑 주소나 계정 식별자가 포함되지 않습니다. 조회되는 정책 규칙·ABI는 실행 코드가 아니라, 서명·스키마 검증을 거쳐 본 확장 프로그램에 번들된 엔진이 해석하는 선언적 데이터입니다(코드로 실행되지 않음).
  • 로그인 시 서버 조회 (선택): 사용자가 Dambi 계정에 로그인한 경우에 한해, 거래 정보(지갑 주소, 해석된 거래 의도)가 Dambi 서버로 전송됩니다. Dambi 서버는 이를 바탕으로 제재 주소·시세·유동성·온체인 상태(잔고 등 읽기 호출) 등 부가 위험 정보를 조회하고, 대시보드 기록에 사용합니다. 로그인하지 않으면 이러한 서버 연동 없이 기기 내 판정만 동작합니다.
  • HyperLiquid 주문 평가 (제3자에 직접 전송): HyperLiquid 주문을 평가할 때마다, 본 확장 프로그램이 Dambi 서버를 거치지 않고 사용자의 HyperLiquid 지갑(master) 주소와 주문 컨텍스트(자산 등)를 HyperLiquid 공개 /info API로 직접 전송해 실시간 레버리지·마진 정보를 가져옵니다. 이 /info는 인증 없는 공개 엔드포인트로, 주소는 매 요청에 POST 파라미터로 명시 전달됩니다(쿠키·세션 기반 식별이 아님). 이는 연결된 지갑으로 HyperLiquid 주문이 감지될 때 발생하며, Dambi 로그인 여부와 무관합니다(사용자가 이미 거래 중인 거래소). 이것이 ‘Dambi 서버 경유’ 방식의 유일한 예외입니다.
  • IP 주소(부수적): 위 조회는 HTTPS 요청이므로 대상 서버가 사용자의 IP를 헤더로 보게 됩니다. 이는 모든 웹 통신에 따르는 부수적 노출이며, Dambi는 IP를 위치 정보로 수집·이용하지 않습니다(지오로케이션 API 미사용).

5. 계정 로그인 (선택)

Dambi 계정 로그인은 선택 사항이며, 클라우드 대시보드와 정책 동기화 기능에만 필요합니다. 로그인 시 Google 인증(chrome.identity)을 통해 이메일 주소를 받아 계정을 식별하고, 등록한 지갑 주소·라벨과 거래 내역을 대시보드 조회용으로 서버에 저장합니다. 로그인하지 않아도 핵심 보안 분석 기능은 완전히 동작합니다.

6. 제3자 데이터 제공자

  • Dambi 서버 경유 (로그인 시에만): 위험 평가가 필요할 때 Dambi 서버가(브라우저가 아니라) 대신 다음 제공자에 관련 주소를 조회합니다. 브라우저는 이 제공자들에 직접 접속하지 않으며, 제공자에게는 Dambi 서버가 노출됩니다. 사용자의 이메일·인증 토큰은 이 제공자들과 공유되지 않습니다.
    • Chainalysis (제재 주소 스크리닝)
    • Alchemy (NFT 시세·블록체인 RPC)
    • GeckoTerminal (풀 유동성)
    • GoPlus (주소 평판)
    • CoinGecko (토큰 메타데이터)
    • Chainlink (시세 오라클)
  • 브라우저에서 직접 (로그인 무관) — HyperLiquid: §4의 설명대로, HyperLiquid 주문 평가 시 사용자의 HyperLiquid 주소·주문 컨텍스트가 HyperLiquid 공개 /info API로 직접 전송됩니다.

각 제공자의 데이터 처리는 해당 제공자의 개인정보 방침을 따릅니다.

7. 데이터를 판매·추적하지 않음

  • Dambi는 사용자 데이터를 판매하지 않으며, 제3자에게 마케팅 목적으로 양도하지 않습니다.
  • 제3자 애널리틱스나 행동 추적을 사용하지 않습니다.
  • 사용자 데이터를 신용도 평가나 대출 목적으로 사용하지 않습니다.
  • 수집된 데이터는 본 확장 프로그램의 단일 목적(서명 전 보안 분석)과 사용자 본인의 대시보드 조회에만 사용되며, 행동 프로파일링·ML 학습·교차 지갑 상관 분석·네트워크 모니터링에 사용하지 않습니다. 운영 로그(타이밍·오류)는 진단 목적에 한합니다.

8. 데이터 보관 및 삭제

  • 판정·실행 기록은 기기에 로컬 저장되며, 확장 프로그램을 제거하면 함께 삭제됩니다.
  • 로그인하지 않은 사용은 서버에 어떠한 데이터도 저장하지 않습니다.
  • 로그인 사용자의 서버 저장 데이터(계정·지갑 목록·거래 내역)는 계정이 활성인 동안 보관되며, 계정 삭제 요청 시 삭제됩니다.
